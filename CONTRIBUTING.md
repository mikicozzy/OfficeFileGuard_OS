# Contributing

Thank you for your interest in contributing to OfficeFileGuard.

Contributions of all kinds are welcome, including:

* Bug reports
* Feature requests
* Documentation improvements
* Bug fixes
* Performance improvements
* Pull requests

## Before Opening an Issue

Please check whether the problem has already been reported.

When reporting a bug, include as much information as possible, such as:

* OfficeFileGuard version
* Windows version
* Microsoft Office version
* Steps to reproduce the issue
* Expected behavior
* Actual behavior

## Pull Requests

If you would like to contribute code:

* Keep changes focused on a single topic.
* Follow the existing coding style whenever possible.
* Add or update unit tests when appropriate.
* Ensure the solution builds successfully before submitting a pull request.

Large architectural changes should be discussed in an Issue before implementation.

## Questions and Suggestions

Suggestions for improving OfficeFileGuard are always appreciated.

If you are unsure whether a feature fits the project's goals, feel free to open an Issue before starting development.

Thank you for helping improve OfficeFileGuard.
